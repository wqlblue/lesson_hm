<!-- package.json版本号特点
package.json中有依赖包版本号，但它常常是有范围的。像 “^” 或 “~” 开头的版本号表示一个范围，这样在这个范围内的版本都可能被安装。这种灵活性在开发时能自动适应小更新，但也有隐患。
锁定文件的作用
锁定文件能记录实际使用的依赖包的具体版本和详细信息。它确保在不同环境下，安装的都是和开发环境一样的依赖包，避免因版本差异导致项目故障。 
所以需要 pnpm-lock.yaml
-->

 ~1.0.0允许更新的范围是保持主版本号和次版本号不变，补丁版本号可以更新。 -->
