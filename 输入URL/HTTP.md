# HTTP 2.0 
 - 1.1的问题：带宽利用率低
   1. TCP的慢启动
   2. 同时开启的多条TCP连接相互竞争带宽  

- 多路复用
1. 打造了一个二进制分帧层，将多个请求利用二进制分帧层处理成一帧一帧的请求，每一帧带上相应的ID编号，服务器接受所有帧后，会根据ID相同的合并成一个完整的请。服务器响应该请求，将响应头和响应体发送至二进制分帧层，浏览器接受到响应帧后自行合并成完整的响应体
2. 多路复用中可以给重要的资源加标记，服务器优先响应权重高的资源
3. 通知只能最多开启一条TCP连接

# HTTP3.0
  - 2.0 的问题：
    1. TCP的队头阻塞:主要因为超时重传，在传输数据包的过程中，一旦出现某个包丢失，后续的所有数据包都要等待丢失的数据包重新打包重发，后续包才能继续发送
    2. TCP建立连接的延时

 - QUIC协议 = TCP + UDP
    1.实现了TCP的流量拥塞控制
    2.集成了TLS加密
    3.多路复用
    4.快速握手
